Problem 2: Indexing Documents via Hadoop

Overview
Problem 2 is implemented as a two-stage MapReduce pipeline:

1. Problem 2(a): Compute document frequency (DF) for each term.
2. Problem 2(b): Use the top 100 DF terms to compute per-document TF-based scores.

The smaller dataset Wikipedia-50-ARTICLES.tar may be used for testing.
All final experiments should be run on Wikipedia-EN-20120601_ARTICLES.tar.gz.

Files used
- problem_2/src/DocumentFrequency.java
- problem_2/src/TfIdfScore.java
- problem_2/src/WholeFileInputFormat.java
- problem_2/src/WholeFileRecordReader.java
- problem_2/src/TextProcessingUtils.java
- data/stopwords.txt
- lib/opennlp-tools-1.9.3.jar

Input format
- Each Wikipedia article must be available as a separate .txt file in an extracted directory.
- Each mapper reads one whole article as one document.

Problem 2(a): Document Frequency
Goal:
Compute TERM<TAB>DF where DF is the number of distinct documents containing the term.

Processing:
1. Tokens are lowercased and cleaned.
2. Stopwords are removed.
3. Porter stemming is applied.
4. Each term is emitted at most once per document.
5. Reducer sums counts to obtain DF.

Run example on the small dataset:
hadoop jar problem_2/jars/document_frequency.jar DocumentFrequency data/Wikipedia-50-ARTICLES problem_2/results/output_problem_2a data/stopwords.txt

Run example on the full dataset:
hadoop jar problem_2/jars/document_frequency.jar DocumentFrequency data/Wikipedia-EN-20120601_ARTICLES problem_2/results/output_problem_2a_full data/stopwords.txt

Output:
problem_2/results/output_problem_2a/part-r-00000
or
problem_2/results/output_problem_2a_full/part-r-00000

Generate the top 100 DF terms
Use the output of Problem 2(a):

Small dataset example:
sort -k2,2nr problem_2/results/output_problem_2a/part-r-00000 | head -100 > problem_2/results/top100_df.tsv

Full dataset example:
sort -k2,2nr problem_2/results/output_problem_2a_full/part-r-00000 | head -100 > problem_2/results/top100_df.tsv

Problem 2(b): TF-based Score per Document
Goal:
For each document and each of the top 100 terms, compute:
SCORE = TF * log(10000 / DF + 1)

Output format:
ID<TAB>TERM<TAB>SCORE

Processing:
1. Load top100_df.tsv from distributed cache.
2. For each document, count TF only for the selected top 100 terms.
3. Compute the score using the DF value from top100_df.tsv.
4. Write the final TSV output.

Run example on the small dataset:
hadoop jar problem_2/jars/tfidf_score.jar TfIdfScore data/Wikipedia-50-ARTICLES problem_2/results/output_problem_2b data/stopwords.txt problem_2/results/top100_df.tsv

Run example on the full dataset:
hadoop jar problem_2/jars/tfidf_score.jar TfIdfScore data/Wikipedia-EN-20120601_ARTICLES problem_2/results/output_problem_2b_full data/stopwords.txt problem_2/results/top100_df.tsv

Output:
problem_2/results/output_problem_2b/part-r-00000
or
problem_2/results/output_problem_2b_full/part-r-00000

Suggested workflow for the full dataset
1. Extract Wikipedia-EN-20120601_ARTICLES.tar.gz
2. Run Problem 2(a)
3. Sort output and create top100_df.tsv
4. Run Problem 2(b)
5. Save runtime outputs and screenshots for the report
